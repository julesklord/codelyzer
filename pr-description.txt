🎯 **What:**
Fixed a Cross-Site Scripting (XSS) vulnerability in the Mermaid diagram rendering logic within `src/App.jsx`.

⚠️ **Risk:**
By directly injecting unsanitized SVG strings (which may be user-derived or manipulated repository data) via `innerHTML`, an attacker could potentially execute arbitrary JavaScript in the context of the user's session.

🛡️ **Solution:**
Wrapped the `result.svg` output from the Mermaid renderer in a `DOMPurify.sanitize()` call before assignment to `innerHTML`. This ensures that malicious tags (like `<script>` or event handlers) are stripped out. Standard DOMPurify configuration is used so that the `<foreignObject>` HTML elements required by Mermaid.js for rendering layout and text are preserved without breaking diagram styles.
